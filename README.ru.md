# Ostrakon

Telegram-бот для **демократического бана спамеров** в группах: участники голосуют за
кик, бот применяет результат. Чистый PHP, без фреймворков, рассчитан на обычный
shared-хостинг.

English version: [`README.md`](README.md).

> **Статус: функционально готов, ждёт боевого тестирования.** Готово: инфраструктура,
> асинхронный вебхук, голосования (режимы light и full с рейтингом активности и статусом
> «аксакал»), многоязычная веб-панель с персональным контролем доступа, личные уведомления
> в ЛС (с командами-ответами), все cron-задачи и экспорт/импорт JSON (`Exporter`) для
> переноса группы между инстансами.

## Немного истории

Название **«Остракон»** отсылает к классическим Афинам V века до н. э., где существовала
процедура *остракизма* (*ostrakismós*): народное собрание могло на десять лет изгнать из города
человека, чьё влияние казалось опасным для демократии. Голосовали, царапая имя изгоняемого на
черепке разбитой глиняной посуды — такой черепок и называется *остракон* (*óstrakon*). Изгнание
не было наказанием в привычном смысле: человек не терял ни гражданства, ни имущества и через
десять лет мог вернуться. Этот бот — цифровой остракон (черепок для голосования): сообщество так
же коллективно решает, кому не место в чате.

## Требования к хостингу (кратко)

| Компонент      | Требование                                                                                     |
|----------------|------------------------------------------------------------------------------------------------|
| PHP            | **8.1+** (разработка на 8.2)                                                                   |
| Расширения PHP | `pdo_mysql`, `curl`, `json` (обязательно); `mbstring` (желательно)                             |
| СУБД           | MySQL 5.7+ **или** MariaDB 10.x (InnoDB, utf8mb4)                                              |
| Веб-сервер     | Apache (`.htaccess` в комплекте) или Nginx (каталоги закрыть вручную) + `mod_rewrite`          |
| HTTPS          | **Обязателен** — вебхук и Login Widget работают только по HTTPS                                |
| Cron           | Возможность запускать `cron.php` по расписанию (раз в минуту — идеально; реже — тоже работает) |

`fastcgi_finish_request` **не требуется**: вебхук асинхронный. Полная инструкция по установке и
настройке — в разделе **[Для хостера бота](docs/for-hosters.ru.md)**.
